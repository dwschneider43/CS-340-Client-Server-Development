## Module Eight Journal Reflection
CS-340 Client/Server Development  
Portfolio Submission - Project Two

For this course, I added my final dashboard code and the README from Project Two to my GitHub portfolio. This project pulled together everything from building the CRUD Python module to creating a working dashboard connected to MongoDB. Here’s my reflection on what I learned and how I approached the work.

## How I write programs that are maintainable, readable, and adaptable
Throughout the course, especially when building the CRUD Python module in Project One, I focused on writing code that was clean and easy to reuse. Keeping all the database operations in one module made the whole project easier to manage. When I moved into Project Two, I didn’t have to rewrite anything I just imported the CRUD functions and connected them to the dashboard widgets. This made the dashboard more reliable and easier to update. If I ever need to build another dashboard or any app that talks to a database, I can reuse this same CRUD module and just adjust it for whatever data I’m working with.

## How I approach problems as a computer scientist
When I started working on the Grazioso Salvare requirements, I approached it by breaking the problem down into smaller pieces. I looked at what the client needed, what the data looked like, and what the dashboard had to show. From there, I built the queries, filters, and visualizations step by step. Compared to earlier courses, this project required more independence and more real-world thinking. I had to design the layout, decide what data mattered, and make sure everything worked together. In the future, I’d use the same approach: understand the client’s goals first, then design the database or dashboard around those goals, and build modular code that can grow as requirements change.

## What computer scientists do, and why it matters
Computer scientists solve problems by building tools and systems that help organizations work more efficiently. In a project like this, the dashboard I created would help a company like Grazioso Salvare quickly analyze rescue dog data and make decisions faster. Instead of digging through raw data, they get instant insights through visualizations and filters. This kind of work matters because it turns data into something useful and helps companies do their jobs better. It showed me how the tools we build can directly support a company’s mission.
